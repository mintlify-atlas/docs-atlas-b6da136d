## GET /memory/{memory_id}

Retrieve a single memory by its unique identifier. This endpoint is useful for fetching detailed information about a specific memory.

### Path Parameters

<ParamField path="memory_id" type="string" required>
  The unique identifier of the memory to retrieve.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates whether the request was successful.
</ResponseField>

<ResponseField name="memory" type="object">
  The memory object with all its details.
  
  <Expandable title="Memory Object">
    <ResponseField name="id" type="string">
      Unique identifier for the memory.
    </ResponseField>
    
    <ResponseField name="memory" type="string">
      The stored memory fact.
    </ResponseField>
    
    <ResponseField name="user_id" type="string">
      The user ID this memory belongs to.
    </ResponseField>
    
    <ResponseField name="metadata" type="object">
      Associated metadata including memory_type, timestamps, and custom fields.
      
      <Expandable title="Metadata Fields">
        <ResponseField name="memory_type" type="string">
          Classification of the memory (LONG_TERM, SHORT_TERM, EPISODIC, SEMANTIC, or PROCEDURAL).
        </ResponseField>
        
        <ResponseField name="source" type="string">
          Where the memory came from (e.g., "conversation", "screen_capture").
        </ResponseField>
        
        <ResponseField name="custom_fields" type="any">
          Any custom metadata fields added when creating the memory.
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="hash" type="string">
      Hash of the memory content for deduplication.
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      ISO timestamp when the memory was created.
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      ISO timestamp when the memory was last updated.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl http://localhost:8000/memory/mem_abc123
```

### Example Response

```json
{
  "success": true,
  "memory": {
    "id": "mem_abc123",
    "memory": "User prefers TypeScript over JavaScript for new projects",
    "user_id": "user_123",
    "metadata": {
      "memory_type": "LONG_TERM",
      "source": "conversation",
      "confidence": "high"
    },
    "hash": "a1b2c3d4e5f6",
    "created_at": "2025-03-01T10:00:00Z",
    "updated_at": "2025-03-01T10:00:00Z"
  }
}
```

### Error Response

If the memory ID doesn't exist:

```json
{
  "detail": "Memory not found"
}
```
